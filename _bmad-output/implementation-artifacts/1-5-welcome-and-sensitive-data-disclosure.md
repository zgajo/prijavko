## Story

As a host on first launch,
I want a clear, Croatian-primary explanation of what passport data the app processes, how long it keeps it, and where the policy lives,
so that I give informed consent before granting camera permission and understand the trust premise before entering credentials.

### AC3 — `lib/app/app.dart` — MaterialApp.router with ConsentGate

1. Create `lib/app/app.dart`:
   ```dart
   class PrijavkoApp extends ConsumerWidget {
     const PrijavkoApp({super.key});

     @override
     Widget build(BuildContext context, WidgetRef ref) {
       final router = ref.watch(routerProvider);

       return MaterialApp.router(
         title: 'prijavko',
         theme: buildLightTheme(),
         darkTheme: buildDarkTheme(),
         themeMode: ThemeMode.system,
         routerConfig: router,
         localizationsDelegates: AppLocalizations.localizationsDelegates,
         supportedLocales: AppLocalizations.supportedLocales,
         builder: (context, child) =>
             ConsentGate(child: child ?? const SizedBox.shrink()),
       );
     }
   }
   ```
2. **Why `builder:` for ConsentGate**: `MaterialApp.router` produces a Navigator as its child. Wrapping in `builder:` places ConsentGate between MaterialApp (theme provider) and the Navigator (route tree). This ensures: (a) `Theme.of(context)` works inside `_ConsentLoadingScaffold`, (b) `consentControllerProvider` resolves via the ancestor `ProviderScope`, (c) the Router/Navigator doesn't build until consent resolves — no wasted route evaluation during the ~50ms consent RPC.
3. **Localization delegates**: `AppLocalizations.localizationsDelegates` bundles the generated delegate + `GlobalMaterialLocalizations`, `GlobalWidgetsLocalizations`, `GlobalCupertinoLocalizations`. `supportedLocales` includes `[Locale('hr'), Locale('en')]` (generated from the ARB files).
4. Remove the `TODO(story-1.5):` comment from `lib/app/providers.dart` — this AC fulfills it.
5. Run `dart run build_runner build --delete-conflicting-outputs` to regenerate `providers.g.dart` if `routerProvider` is added to `providers.dart`. If router has its own file, generate its `.g.dart` instead.

## Dev Notes

### Why this story is fifth

Stories 1.1–1.4 built scaffold, design system, security primitives, and UMP consent. Story 1.5 is the **first user-facing screen with project content** (not a SDK form or a design-system smoke preview). It introduces three foundational pieces simultaneously:
1. **go_router** — every subsequent screen depends on routing.
2. **AppLocalizations / ARB** — every subsequent screen depends on localized strings.
3. **Feature-based screen structure** — `lib/features/onboarding/welcome_screen.dart` is the template for all future screens.

The welcome screen is intentionally minimal (static text + links + one button) because it carries infrastructure debt from three new dependencies. Story 1.6 (camera permission) builds on this infrastructure without adding new foundational dependencies.

### Architecture mandates (non-negotiable)

- **Feature-based folders**: `lib/features/onboarding/welcome_screen.dart` — per architecture §Project Structure and figma-code-contract §4 row 01.
- **`@riverpod` codegen only**: `routerProvider` in `router.dart` uses `@Riverpod(keepAlive: true)`. No manual `Provider(...)` calls.
- **`keepAlive: true`** for the router provider — GoRouter holds navigation state for the app's lifetime. Disposing loses the back-stack.
- **No cross-feature imports**: `lib/features/onboarding/` imports from `lib/core/` and `lib/design/` only. `lib/app/router.dart` imports feature screens — this is the router's job, not a cross-feature violation.
- **All UI strings via `AppLocalizations`**: Zero literal strings in `build()`. Croatian primary, English fallback. Full diacritics (č/ć/š/ž/đ — no ASCII approximation). Sentence case. Imperative CTAs.
- **Dark mode primary**: Build and verify dark first. `buildDarkTheme()` is the design target. Light mode inherits from `buildLightTheme()` via `ColorScheme.fromSeed`.
- **`ConsentGate` placement**: Stays as the root-level gate wrapping the navigable tree. Moved from `home:` parameter (Story 1.4) to `builder:` callback (Story 1.5) because `MaterialApp.router` has no `home:` parameter.
- **Standard screen skeleton** from UX spec §Spacing & Layout Foundation — SafeArea, 16dp horizontal padding, bottom CTA zone (72dp = 56dp button + 16dp padding), gesture inset (24dp bottom).
- **No ads on onboarding screens** — per UX spec §Ad Placement and architecture §AdMob Placement Policy.
- **WCAG 2.1 AA contrast**: 4.5:1 body, 3:1 large text. M3 seed-based schemes meet this by default — verify visually.

### LLM-specific anti-patterns for this story

| ❌ Do NOT | ✅ Do THIS instead |
|---|---|
| Use `MaterialApp` with `home:` parameter | Use `MaterialApp.router` with `routerConfig:` — go_router provides the Router |
| Create `GoRouter` as a global `final` variable | Use `@Riverpod(keepAlive: true)` provider — enables test overrides and future redirect logic reading other providers |
| Add auth redirect to the router now | Defer to Story 2.x. Add `// TODO(story-2.3): add redirect callback reading authNotifierProvider` |
| Call `context.push('/onboarding/camera-permission')` | Use `context.go(...)` — linear onboarding has no back-stack. Push creates a stack entry; go replaces. |
| Use `Navigator.push` or `Navigator.of(context)` | Use `go_router`'s `context.go()` / `context.goNamed()`. Navigator 1.0 API is banned once go_router is in the dep tree. |
| Put WelcomeScreen in `lib/core/` | It goes in `lib/features/onboarding/welcome_screen.dart` — it's a feature screen, not infrastructure. |
| Create a custom `HyperlinkText` widget | Use `Text.rich` + `TextSpan` + `TapGestureRecognizer`. No need for a reusable widget — only two screens use inline links (Welcome + "Your Data" in Story 8.1). |
| Use `RichText` widget directly | Use `Text.rich` — it inherits `DefaultTextStyle` from the theme. `RichText` does not, causing style reset. |
| Hardcode colors for link text | Use `Theme.of(context).colorScheme.primary` for link color. |
| Forget to dispose `TapGestureRecognizer` | Dispose in `State.dispose()`. Leaked recognizers hold references to the widget tree. |
| Use `ConsumerWidget` for a widget that creates/disposes gesture recognizers | Use `ConsumerStatefulWidget` — recognizers need lifecycle management. |
| Skip `AppLocalizations` and use raw strings "for now" | Every string from day one goes through ARB. No shortcuts. |
| Use `intl: ^0.19.0` (pinned) | Use `intl: any` — let `flutter_localizations` resolve the version. Pinning causes constraint conflicts. |
| Put the camera-permission screen implementation in this story | Create only the placeholder route. Story 1.6 implements the real screen. |
| Move `applyMainAppFontConfig()` to app.dart | Keep it in main.dart — `offline_fonts_test.dart` imports it from there. |
| Add `import 'package:url_launcher/...'` outside `lib/features/onboarding/` | Only welcome_screen.dart (and later your_data_screen.dart in Story 8.1) use url_launcher. Keep imports scoped. |
| Skip golden tests | The epic AC explicitly requires "golden tests exist for both dark and light variants". Generate with `--update-goldens`. |
| Use `AppLocalizations.of(context)!` (with `!`) | `l10n.yaml` sets `nullable-getter: false`, so `of(context)` returns non-null. No `!` needed. |
| Import `package:flutter_localizations/flutter_localizations.dart` in widget files | Only `app.dart` needs the delegates import. Widgets import `AppLocalizations` only. |

### url_launcher — version 6.3.2

- `launchUrl(Uri.parse('https://...'), mode: LaunchMode.externalApplication)` is the API. Returns `Future<bool>` — true if launched.
- Android-only project — no iOS `Info.plist` LSApplicationQueriesSchemes needed.
- No special permissions required for https URLs on Android.
- `canLaunchUrl()` is NOT needed for https URLs — Android always has a handler. Skip the check for simplicity.

### go_router — version 17.2.2

- `MaterialApp.router(routerConfig: goRouter)` is the integration point.
- `context.go('/path')` replaces the entire navigation stack up to the matched route. `context.push('/path')` adds to the stack.
- For linear onboarding (no back), use `context.go` — not `context.push`.
- Route paths are kebab-case: `/onboarding`, `/camera-permission`, `/login`.
- `GoRoute.name` is used for `goNamed()` — optional but useful for refactoring resilience.

### Flutter l10n — gen-l10n setup

- `l10n.yaml` drives `flutter gen-l10n`.
- Generated files go to `.dart_tool/flutter_gen/gen_l10n/` — NOT committed (regenerated automatically with `generate: true`).
- Import path: `package:flutter_gen/gen_l10n/app_localizations.dart` (synthetic package).
- `nullable-getter: false` means `AppLocalizations.of(context)` returns `AppLocalizations`, not `AppLocalizations?`. Cleaner call sites.
- English is the template (defines keys + descriptions). Croatian is the primary runtime locale.
- `supportedLocales` is auto-generated from the ARB files: `[Locale('en'), Locale('hr')]`. The order depends on the ARB file names — `app_en.arb` and `app_hr.arb` → `en` and `hr`.
- To make Croatian the default for Croatian device locales, `supportedLocales` list order doesn't matter — `localeResolutionCallback` resolves based on device locale. A Croatian device gets `hr`, an English device gets `en`.
