## Deep story analysis (themes)

### Recurring struggles and solver reality

- **Pub dependency graph:** Story 1.1 documented that `riverpod_generator` and `riverpod_lint` were not added due to solver conflicts with `flutter_test` / `drift_dev`. The team used hand-written `Provider` for `AppConfig` and deferred full lint stack to a follow-up. This is a **systems** issue (toolchain constraints), not individual fault.  
- **Generated code review:** Story 1.2 deferred re-verification of `app_database.g.dart` under review rules that excluded `*.g.dart`. Risk: schema drift vs intent until explicitly verified.  
- **Cross-story bundling:** Code review flagged **same commit** bundling theme work with 1.4/1.5 stubs and `epics.md` edits — noise for reviewers and harder bisect.  

### Recurring review feedback

- **Fail-safe behavior:** 1.5 review caught `ConnectivityBanner` hiding on `AsyncValue.error` (plugin failure looked like “online”). Patched to show offline strip — good example of **poka-yoke** for user-visible connectivity.  
- **Stream robustness:** `watchAllFacilities` needed `onError` on the refresh notifier so router refresh does not throw uncaught async errors.  
- **Strict validation:** `parseYyyyMmDd` calendar validation tightened after review (1.2).  
- **Theme semantics:** `sent` vs `pausedAuth` colors distinguished after review (1.3).  

### Breakthrough patterns worth repeating

- **Drift watch → go_router refresh:** `FacilitiesRouteRefreshNotifier` + `GoRouter.refreshListenable` gives Epic 2 a clean hook when facilities change — no polling.  
- **Test discipline with Drift:** In-memory DB + `closeStreamsSynchronously` + explicit `db.close()` in tests avoided flaky stream/timer issues (1.5 notes).  
- **Deferred work list:** Central `deferred-work.md` captures low-risk follow-ups without blocking story closure.
