prijavko — Design Direction Exploration

Three focused design directions for prijavko v1.0, each with 8 full-flow screens. Login is username + password only — the eVisitor API key is embedded at build time. The authentication + facility-picker flow is shared across all directions; the 6 post-login screens express each direction's bet on layout density, primary-action posture, and queue visibility. All mockups render with the locked Adriatic Teal + Manrope + Material 3 tokens.

Direction A — Calm & Spacious (recommended baseline)

Bet: single big CTA, generous whitespace, one thing to do per screen. Empty-state home is minimal; when queue has guests, their rows appear inline. Multi-guest batch + explicit Send All preserved unchanged.

Chosen baseline: Direction A + B's inline queue + B's manual-entry CTA

Core pattern: Direction A (calm, single-CTA, minimal chrome) as the visual and interaction baseline. When queue has unsent guests, home shifts to B's inline-rows layout. Manual-entry is always a visible secondary CTA (not a hidden fallback) — it's a real path for non-EU IDs, worn documents, and denied camera permissions.

Login flow: username + password only. The eVisitor API key is embedded in the app at build time (single shared key across all users — confirmed with HTZ in Week-1 spike). Known residual risk: APK extraction could expose the key; mitigated by Play Store update path (~3-day rotation) and the likely coarse-grained nature of the vendor API key.

Ad placement (v1.0): single AdMob anchored-adaptive banner (50–100dp) on Home only, between queue content and CTA section. Never on onboarding, login, facility picker, scan, Send All, credential banner, reauth, closure summary, or "Your Data." No interstitials — preserves the trust thesis and the sacred Closure Summary moment. Pre-agreed pivot (from PRD): if reviews flag ads as #1 complaint, pivot to ad-free free tier + earlier v1.1 Pro unlock.

Direction C deferred to v1.1 as an optional setting ("Always-on camera on home"), only if real demand surfaces post-launch. No speculative build cost in v1.0.
